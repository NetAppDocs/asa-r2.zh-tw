[cols="2,4"]

[[security]]
== 安全性

[cols="2,4"]
|===
| 更新 | 說明 


| link:../secure-data/enable-anti-ransomware-protection.html["支援具有 SnapMirror 主動式同步關係的 ARP/AI"] | 人工智慧自主勒索軟體防護 (ARP/AI) 支援 ASA r2 系統上處於 SnapMirror 主動同步關係中的主讀寫儲存設備單元。ARP/AI 在活動主儲存設備單元上運行，建立僅針對主儲存設備單元的 ARP 快照，並為這些儲存設備單元提供基於機器學習的勒索軟體偵測。先前因同步複寫關係而導致預設 ARP 能力開發受到抑制的叢集升級至 ONTAP 9.19.1 後，ARP 將自動恢復。更新前警告會通知管理員，並提供停用 ARP 的選項。 


| link:../learn-more/additional-supported-ontap-features.html#tls-hardware-offload["支援 TLS 硬體卸載"] | 您可以將整個叢集的傳輸層安全性（TLS）加密和解密任務卸載到支援的乙太網路卡上。這可以降低 CPU 開銷，並提高使用 TLS 的系統的效能。 
|===


[[storage-efficiency]]
== 儲存效率

[cols="2,4"]
|===
| 更新 | 說明 


| link:../learn-more/additional-supported-ontap-features.html#global-deduplication["支援全域重複資料刪除"] | 全域重複資料刪除功能將現有的 ONTAP 重複資料刪除空間節省範圍擴展到整個儲存可用區（SAZ）。透過消除 SAZ 中所有儲存單元的重複資料區塊，它可以顯著提高儲存效率並降低整體擁有成本。全域重複資料刪除功能預設為啟用，無需任何管理配置即可透明運行。 
|===

| link:../learn-more/additional-supported-ontap-features.html#modify-default-settings-that-control-automatic-qos-throughput-ceiling-increases["支援修改控制自動 QoS 吞吐量上限增加的預設設定"] | 您可以修改預設設定，這些設定控制 QoS 原則群組如何根據短期效能需求自動提高 QoS 處理量上限。這些設定在 ONTAP 9.18.1 及更早版本中無法修改。
